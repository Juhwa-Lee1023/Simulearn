1.2 서비스 소개

이 서비스의 핵심은 **'상황의 맥락(Context)'**과 **'페르소나의 일관성'**입니다.
왼쪽 메인 창: AI 동료와의 메신저(Slack/Teams UI) 및 문서 편집기.
오른쪽 사이드바 (LMS 역할): * Quick Tips: 현재 상황에서 필요한 이론(예: PRD 작성법, 예외 케이스 정의법).
진척도: 전체 프로젝트 단계 (기획 -> 디자인 -> 개발 -> QA -> 배포).

1.3 목표 및 목적

핵심 목표:
 - 직무 맥락을 유지한 채 AI 동료 피드백을 빠르게 받는 협업 시뮬레이션 제공
 - 메신저, 문서, 학습 팁을 한 화면에 통합해 피드백 루프를 단축
 - 멘탈 게이지/엔딩/리포트로 학습 동기와 자기 점검 강화

타겟 사용자:
 - IT 서비스 기획자(프로토타입 기본 직무)
 - 팀 내 리뷰를 빠르게 받고 싶은 주니어/주니어-미들 실무자

해결하려는 문제:
 - 산출물 작성 중 맥락이 분산되어 피드백 품질이 낮아지는 문제
 - 예외 케이스/리스크 점검이 늦어지는 문제
 - 반복 피드백 과정에서 동기/멘탈이 쉽게 소진되는 문제

2.2 기능 상세 설명
기능 1: 가상 협업 환경 대시보드

목적:
 - 메신저와 문서 편집을 동시에 제공해 맥락 단절을 최소화
 - 학습 팁과 진행상태를 실시간으로 가시화

사용자 시나리오:
1. 사용자가 직무/난이도를 선택하고 대시보드에 진입한다.
2. 상사 메시지가 도착하고 사용자는 문서 초안을 작성한다.
3. 오른쪽 Quick Tips와 진행도 상태를 보며 보완한다.

주요 요구사항:
 - 왼쪽 메신저: 메시지 카드 목록 (상사/페르소나/사용자 구분)
 - 왼쪽 문서: Markdown 텍스트 영역 (초안 작성)
 - 오른쪽 Quick Tips: Step 1/2/3 별 정적 팁 노출
 - 오른쪽 진척도: 기획→디자인→개발→QA→배포 단계 표시
 - 상태 복원: 로컬 저장소로 단계/메시지/문서/멘탈 상태 복원

기능 2: AI 동료 페르소나 (Multi-Agent)

목적:
 - 직무별 관점에서 날카로운 질문을 던지는 피드백 제공

사용자 시나리오:
1. 사용자가 피드백 요청 버튼을 누른다.
2. 시니어 개발자 → 디자이너 → 사업 리더 순으로 응답한다.
3. 각 응답은 메신저에 스트리밍으로 표시된다.

주요 요구사항:
 - 페르소나 3종: 시니어 개발자(엣지 케이스), 프로덕트 디자이너(디자인 시스템), 사업 리더(일정/리스크)
 - 순차 응답: 한 명씩 순서대로 피드백
 - Claude API 사용: 서버 프록시(`/api/claude`)에서 스트리밍 응답
 - 요청 컨텍스트: 직무/난이도 + 문서 초안 포함

페르소나역할피드백 스타일시니어 개발자기획 리뷰어"이 로직에서 Edge Case(예외 상황) 처리가 빠졌네요. 데이터가 없을 땐 어떻게 보여주나요?"UI/UX 디자이너시각화 파트너"기획하신 기능이 현재 디자인 시스템 가이드라인과 맞지 않아요. 버튼이 너무 많습니다."사업부 담당자요구사항 제안자"이 기능 다음 주까지 안 되나요? 당장 마케팅 이벤트 시작해야 하는데요."

기능 3: 시뮬레이션 단계 (User Flow)

목적:
 - 단계형 진행으로 사용자의 집중과 피드백 루프를 설계

사용자 시나리오:
1. Step 1에서 직무/난이도 선택
2. Step 2에서 상사 메시지로 미션 투척
3. Step 3에서 피드백 요청 → 멘탈 게이지 반영 → 종료/리포트

주요 요구사항:
 - Step 1: 기본 직무는 IT 서비스 기획자, 난이도 2종
 - Step 2: 미션 메시지 자동 삽입
 - Step 3: 피드백 요청 버튼, 진행 중 상태 표시
 - 멘탈 게이지: 피드백 응답 후 감소, 커피 타임 +20 (1회)
 - 엔딩: 멘탈/루브릭 기준으로 성과급/야근
 - 최종 리포트: 강점/보완점/페르소나 요약

3. 사용자 플로우
3.1 메인 플로우

[시작] → Step 1. 직무 및 난이도 선택 → Step 2. 미션 투척 (Trigger) → Step 3. 실제 산출물 작성 및 피드백 (Interactive Loop) → [완료]

3.2 상세 플로우

각 주요 기능별 사용자 여정을 단계별로 작성

시뮬레이션 단계 (User Flow) 플로우

Step 1: 직무 및 난이도 선택
사용자가 'IT 서비스 기획자' 선택 -> '신입 사원 온보딩' 혹은 '장애 대응 상황' 등 난이도 선택.

Step 2: 미션 투척 (Trigger)
메신저 알림음과 함께 상사가 메시지를 보냅니다.
"OO 님, 이번에 '장바구니 쿠폰 자동 적용' 기능 건으로 개발팀이랑 회의 있는데, 그 전까지 초안 기획서 좀 공유해 주세요."

Step 3: 실제 산출물 작성 및 피드백 (Interactive Loop)
사용자가 기획서를 작성하고 "피드백 요청"을 누르면, 페르소나 3명이 순차적으로 응답합니다.
AI: "쿠폰 유효기간이 만료되는 시점에 결제가 일어나면 어떻게 처리할까요?"
사용자: (답변 작성 또는 수정)
오른쪽 팁: "기획자는 'Happy Path'뿐만 아니라 'Abnormal Path'까지 정의해야 합니다."라는 가이드 노출.
멘탈 게이지: 피드백 라운드 후 감소, 커피 타임으로 +20 회복 (1회)
종료: "시뮬레이션 종료" 클릭 시 리포트/엔딩 생성

[기능 2] 플로우

Step 1: 피드백 요청
 - 사용자가 피드백 요청 버튼 클릭
 - 활성 페르소나 상태 표시

Step 2: 스트리밍 응답
 - Claude API 스트리밍 응답을 메신저에 실시간 누적
 - 페르소나별 메시지 카드 생성

Step 3: 점수/멘탈 반영
 - 루브릭 점수(목표/핵심/예외/일정 키워드) 계산
 - 멘탈 게이지 업데이트 및 엔딩 조건 반영

💡 차별화 포인트 (Witty Add-ons)

'멘탈 관리' 게이지: 피드백 라운드 후 멘탈이 하락하며, 커피 타임으로 1회 +20 회복.
멀티 시나리오: 멘탈 >= 70 그리고 루브릭 >= 70이면 '성과급', 그 외는 '야근' 엔딩.
최종 리포트: 강점/보완점/페르소나 피드백 요약 카드 제공.
